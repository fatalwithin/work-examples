```
.env --файл для сбора всех переменных, secret-ов, должен быть в .gitignore

pipenv --папка, внутри которой собраны зависимости pip-пакетов, которые нужны для проекта
|
└───/requirements.txt

app --корневая папка проекта
|
└───/settings.py --настройки на уровне проекта, можно сделать отдельным модулем, dotenv(конфликтует ли с pipenv?), decouple
|
└───/urls.py --ссылки
|
└───/constants.py --константы
|
└───/logging.py --словари для логирования, для импорта в settings.py
|
└───/celery.py --асинхронные задачи
|
└───/config-parser.py --сохранение переменные в .env

common --независимые ни от чего ресурсы
|
└───/models.py --абстрактные классы и модели: id->uuid, createdAt, updatedAt
|
└───/utils.py --утилиты - функции и классы общего назначения,
|
└───/views.py --mix-in, views(представления)
|
└───/serializers.py --сериализаторы

user --номер телефона нельзя делать unique
|
└───/models.py --наследование от AbstractBaseUser
|
└───/soft_delete --"батарейка" - подключаемый модуль для нового функционала и reuse кода
|
└───/receivers.py --сигналы django - события для модели
|
└───/tasks.py

docker --файлы, связанные с запуском сервисов локально в docker-compose для разработки и тестирования
|
└───/docker-compose.yml --файл для локального развертывания 
|
└───/run_app.sh --аргументы: 
        | -migrate --миграция схемы БД из одной структуры в другую либо вместе с данными - на основе модели(3)
        | -downgrade --откатить базу после миграции
        | -collect --сбор статических файлов (CSS итд) (4)
        | -gunicorn --подъем веб-сервера
        | -test --юнит-тесты (2)
        | -lint --подключение линтера (1)
        | -dev_server --
    |
    └───/Dockerfiles/
        |... --докерфайлы для запуска сервисов, которые подтягиваются в docker-compose
```
